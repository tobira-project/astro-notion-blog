# blog_accountsテーブル設計承認依頼
## 設計方針
1. **accountsテーブルと独立管理**
   - accountsが削除されてもblog_accountsは残る
   - account_idがnullの場合は無効として扱う

2. **Stripe決済対応**
   - サブスクリプション管理
   - 支払い履歴追跡

## メインテーブル: blog_accounts

### 基本情報
| カラム名 | 型 | 説明 |
|---------|-----|------|
| **id** | String | 主キー |
| **account_id** | String? | tobiratory accountsとの関連（削除されても残る） |

### Stripe連携
| カラム名 | 型 | 説明 |
|---------|-----|------|
| **stripe_customer_id** | String? | Stripe顧客ID |
| **stripe_subscription_id** | String? | サブスクリプションID |
| **subscription_status** | String | active/inactive/canceled/past_due |
| **subscription_plan** | String? | basic/premium/enterprise |

### 有料会員管理
| カラム名 | 型 | 説明 |
|---------|-----|------|
| **is_premium** | Boolean | 有料会員フラグ |
| **premium_level** | Int | 0:無料 1:ベーシック 2:プレミアム |

### トークン（要確認）
| カラム名 | 型 | 説明 |
|---------|-----|------|
| **token_balance** | Int | 保有トークン数 |
| **tokens_earned** | Int | 累計獲得数 |
| **tokens_spent** | Int | 累計使用数 |

**質問**: ブログ閲覧にトークンは不要と理解していますが、将来的なトークンシステムを見据えてカラムを用意しておくべきでしょうか？

## 📋 関連テーブル

### 1 payment_history（支払い履歴）
```
- blog_account_id: 購入者
- amount: 金額
- payment_type: subscription/one_time/token
- payment_status: succeeded/pending/failed
- stripe_payment_intent_id: Stripe決済ID
```

### 2 subscription_plans（プラン定義）
```
- plan_id: basic/premium/enterprise
- price_monthly: 月額料金
- stripe_price_id_monthly: Stripe価格ID
- features[]: プラン特典リスト
- monthly_tokens: 月次付与トークン
```
