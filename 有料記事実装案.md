# TOBIRACAST 有料記事実装案

## 1. Notion区切り線を使った有料部分の識別方法

### ✅ 確認済み：HTML出力形式

Notionの区切り線はHTMLで以下のように出力されます：
```html
<hr class="divider" data-astro-cid-yfmihcec="">
```

### 実装アプローチ A: HTMLパース方式（推奨）

```typescript
// src/lib/premium-content-parser.ts
export function parsePremiumContent(htmlContent: string) {
  // 区切り線のパターン
  const dividerPattern = /<hr\s+class="divider"[^>]*>/i
  
  // 区切り線で分割
  const parts = htmlContent.split(dividerPattern)
  
  if (parts.length === 1) {
    // 区切り線がない場合は全て無料コンテンツ
    return {
      freeContent: htmlContent,
      premiumContent: '',
      hasPremium: false
    }
  }
  
  return {
    freeContent: parts[0],
    premiumContent: parts.slice(1).join('<hr class="divider">'),
    hasPremium: true
  }
}
```

### 実装アプローチ B: 特殊マーカー利用

```typescript
// 特定のテキストマーカーで有料部分を識別
// 例: "---PREMIUM---" や "【有料記事ここから】"
const PREMIUM_MARKERS = ['---PREMIUM---', '【有料記事ここから】', '===有料コンテンツ===']

const identifyPremiumSection = (blocks: NotionBlock[]) => {
  const premiumStartIndex = blocks.findIndex(block => {
    if (block.type === 'paragraph' && block.paragraph?.rich_text) {
      const text = block.paragraph.rich_text.map(t => t.plain_text).join('')
      return PREMIUM_MARKERS.some(marker => text.includes(marker))
    }
    return false
  })
  
  if (premiumStartIndex === -1) return { freeBlocks: blocks, premiumBlocks: [] }
  
  return {
    freeBlocks: blocks.slice(0, premiumStartIndex),
    premiumBlocks: blocks.slice(premiumStartIndex + 1)
  }
}
```

## 3. ユーザーの有料会員判定ロジック

```typescript
// src/lib/premium-check.ts
import { getAuth } from 'firebase/auth'

export async function checkUserPremiumStatus(userId?: string) {
  if (!userId) return false
  
  try {
    // APIからblog_accountsの情報を取得
    const response = await fetch(`/api/blog-accounts/${userId}`)
    const blogAccount = await response.json()
    
    // accountが紐付いていない場合は無効
    if (!blogAccount.account_id) {
      console.log('Account not linked')
      return false
    }
    
    // 有料会員チェック
    return blogAccount.is_premium && 
           blogAccount.subscription_status === 'active'
  } catch (error) {
    console.error('Premium check failed:', error)
    return false
  }
}

// 記事ごとのアクセス権限チェック
export async function canAccessPost(userId: string, postSlug: string) {
  const blogAccount = await getBlogAccount(userId)
  
  // 有料会員は全記事アクセス可能
  if (blogAccount?.is_premium) return true
  
  // 個別購入した記事かチェック
  if (blogAccount?.purchased_posts?.includes(postSlug)) return true
  
  // アクセス可能な記事リストをチェック
  if (blogAccount?.accessible_posts?.includes(postSlug)) return true
  
  return false
}
```

## 7. 代替UI案（区切り線が使えない場合）

### A. カスタムプロパティ利用
- Notionに「PremiumStartBlock」番号プロパティを追加
- 指定したブロック番号以降を有料コンテンツとする

### B. 別ページ分離
- 無料記事と有料記事を別々のNotionページで管理
- リレーションで紐付け

### C. タグベース
- 特定のタグ（#premium）がついた記事全体を有料化
- 部分的な有料化は不可

## 推奨実装

**Notion区切り線（divider）を使った実装を推奨**

理由：
1. 著者が直感的に有料部分を指定できる
2. Notionエディタ上で視覚的に確認可能
3. 実装がシンプル
4. 後から変更が容易
